# 笔记管理模块使用说明

## 📚 功能概述

笔记管理模块是一个功能完善的知识管理系统，支持分类管理和树形目录结构，适用于技术文档、学习笔记等场景。

### 核心功能

1. **笔记分类管理**
   - 支持多个分类（React、Vue、UniApp等）
   - 每个分类可独立配置图标、描述、排序
   - 分类编码唯一性校验
   - 删除分类前检查是否有关联笔记

2. **笔记管理**
   - 树形目录结构（支持无限层级）
   - 支持目录和笔记两种类型
   - Markdown格式支持
   - 标签管理
   - 阅读次数统计
   - 公开/私有权限控制

## 🗄️ 数据库设计

### 1. 笔记分类表（sys_note_category）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| category_id | bigint(20) | 分类ID（主键） |
| category_name | varchar(50) | 分类名称 |
| category_code | varchar(50) | 分类编码（唯一） |
| category_icon | varchar(100) | 分类图标 |
| category_desc | varchar(500) | 分类描述 |
| order_num | int(4) | 显示顺序 |
| status | char(1) | 状态（0正常 1停用） |
| del_flag | char(1) | 删除标志 |

### 2. 笔记表（sys_note）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| note_id | bigint(20) | 笔记ID（主键） |
| parent_id | bigint(20) | 父笔记ID（0表示根节点） |
| category_id | bigint(20) | 分类ID |
| note_title | varchar(200) | 笔记标题 |
| note_content | longtext | 笔记内容（支持Markdown） |
| note_type | char(1) | 笔记类型（1目录 2笔记） |
| note_tags | varchar(500) | 笔记标签 |
| order_num | int(4) | 显示顺序 |
| status | char(1) | 状态 |
| is_public | char(1) | 是否公开 |
| read_count | bigint(20) | 阅读次数 |
| del_flag | char(1) | 删除标志 |

## 🚀 安装步骤

### 1. 执行数据库脚本

```bash
# 在MySQL数据库中执行
mysql -u用户名 -p密码 数据库名 < sql/note_system.sql
```

脚本会自动创建：
- 笔记分类表和笔记表
- 初始化3个分类（React、Vue、UniApp）
- 插入12条示例笔记数据
- 创建菜单权限数据

### 2. 重启后端服务

```bash
# Windows
cd xiaohe-admin
mvn clean package
java -jar target/xiaohe-admin.jar

# 或者直接运行
.\xiaohe.bat
```

### 3. 访问系统

访问 `http://localhost:8080`，使用管理员账号登录后，在左侧菜单中找到"笔记管理"模块。

## 🎯 API接口说明

### 笔记分类接口

```
GET    /note/category/list          # 查询分类列表
GET    /note/category/{categoryId}  # 获取分类详情
POST   /note/category               # 新增分类
PUT    /note/category               # 修改分类
DELETE /note/category/{categoryIds} # 删除分类
POST   /note/category/export        # 导出分类
```

### 笔记接口

```
GET    /note/manage/list                  # 查询笔记列表
GET    /note/manage/treeList              # 查询笔记树形列表
GET    /note/manage/{noteId}              # 获取笔记详情（自动增加阅读次数）
POST   /note/manage                       # 新增笔记
PUT    /note/manage                       # 修改笔记
DELETE /note/manage/{noteIds}             # 删除笔记
POST   /note/manage/export                # 导出笔记
```

## 💡 使用示例

### 1. 创建分类

```json
POST /note/category
{
  "categoryName": "Spring Boot",
  "categoryCode": "springboot",
  "categoryIcon": "🍃",
  "categoryDesc": "Spring Boot框架学习笔记",
  "orderNum": 4,
  "status": "0"
}
```

### 2. 创建目录

```json
POST /note/manage
{
  "parentId": 0,
  "categoryId": 1,
  "noteTitle": "React进阶",
  "noteType": "1",
  "orderNum": 3,
  "status": "0",
  "isPublic": "1"
}
```

### 3. 创建笔记

```json
POST /note/manage
{
  "parentId": 1,
  "categoryId": 1,
  "noteTitle": "React性能优化",

  "noteType": "2",

  "status": "0",
  "isPublic": "1"
}
```

## 🔐 权限配置

系统已自动配置以下权限：

**笔记分类权限**
- `note:category:list` - 查询分类
- `note:category:query` - 分类详情
- `note:category:add` - 新增分类
- `note:category:edit` - 修改分类
- `note:category:remove` - 删除分类
- `note:category:export` - 导出分类

**笔记管理权限**
- `note:manage:list` - 查询笔记
- `note:manage:query` - 笔记详情
- `note:manage:add` - 新增笔记
- `note:manage:edit` - 修改笔记
- `note:manage:remove` - 删除笔记
- `note:manage:export` - 导出笔记

## 🎨 特色功能

### 1. 树形目录结构
- 支持无限层级的目录嵌套
- 自动构建树形数据结构
- 父子节点关联管理

### 2. 智能删除检查
- 删除分类前检查是否有关联笔记
- 删除笔记/目录前检查是否有子节点
- 防止数据孤立

### 3. 阅读统计
- 查看笔记详情自动增加阅读次数
- 可用于热门笔记排序

### 4. 标签系统
- 支持多标签（逗号分隔）
- 便于笔记分类和检索

### 5. 权限控制
- 支持公开/私有设置
- 可扩展用户级权限控制

## ⚠️ 注意事项

1. 笔记内容字段为`longtext`类型，注意数据库字符集配置
2. 删除操作为逻辑删除，不会真正删除数据
3. 建议定期备份笔记数据
4. 阅读次数统计是累加的，不可回退
5. 分类编码一旦创建不建议修改
